# Migration Guide

Changes introduced in the epistemic revision (v5.1) to align the framework,
scoring, and CLI with hypothesis-conditioned evidence rather than opaque
confidence composites.

---

## Scoring: `overall_confidence` → `support_profile`

### What changed

The `DiagnosticScore.overall_confidence: float` field has been replaced by
`support_profile: DiagnosticSupportProfile`. A read-only `overall_confidence`
property is preserved as a compatibility shim — it returns
`support_profile.dominant_score`.

### Why

A single composite confidence score is uninterpretable without knowing which
hypothesis it supports. The new profile reports per-hypothesis evidence scores
alongside a `dominant` hypothesis label and flags for `evidence_thin` and
`multi_causal` outcomes.

### Migration

**Reading reports (JSON):**

```python
# Before (still works via compat property)
score = report["score"]["overall_confidence"]  # float

# After (preferred)
profile = report["score"]["support_profile"]
dominant = profile["dominant"]          # "model-level", "runtime-host",
                                        # "conversation", "multi-causal", or "unknown"
dominant_score = profile["dominant_score"]
hypotheses = profile["hypotheses"]      # list of {name, score, evidence_quality, layer_mentioned}
```

**In Python code:**

```python
from robopsych.scoring import DiagnosticScore, DiagnosticSupportProfile

# Before
score = DiagnosticScore(..., overall_confidence=0.85)  # field no longer exists

# After
from robopsych.scoring import HypothesisSupport
profile = DiagnosticSupportProfile(hypotheses=[
    HypothesisSupport("model-level", 0.85, evidence_quality=0.9, layer_mentioned=True),
    HypothesisSupport("runtime-host", 0.3, evidence_quality=0.3, layer_mentioned=False),
    HypothesisSupport("conversation", 0.2, evidence_quality=0.2, layer_mentioned=False),
])
score = DiagnosticScore(..., support_profile=profile)

# Compat property still works
assert score.overall_confidence == profile.dominant_score
```

### Multi-causal and unknown outcomes

- `dominant == "multi-causal"`: top two hypotheses are within 0.15 of each other.
  Evidence supports more than one layer; do not force a single-layer attribution.
- `dominant == "unknown"`: best hypothesis score < 0.3. Insufficient evidence to
  support any hypothesis. Record the uncertainty and stop.

## Coherence: assessment value rename

### What changed

| Before (v5.0) | After (v5.1) | Meaning |
|---------------|--------------|---------|
| `"genuine"` | `"high-continuity"` | High backward-reference density, low contradiction |
| `"mixed"` | `"partial"` | Some continuity, some contradiction |
| `"performed"` | `"fragmented"` | Low continuity or high-severity contradiction |

### Why

The old terms implied a truth claim about model transparency. The new terms
describe the observable claim-continuity pattern. High continuity may result
from honest behavior, from narrow topic scope, or from pattern-matching to prior
outputs — the analyst must interpret which.

### Migration

**CLI color maps** accept both old and new values for backward compatibility with
existing JSON reports. Old reports opened with `robopsych score` will display
correctly.

**Python code:**

```python
# Before
if report.assessment == "genuine":
    ...

# After
if report.assessment == "high-continuity":
    ...
```

**Calibration reference set:** The committed reference set at
`validation/calibration/reference_set/labels.json` retains the pre-v5.1 label
vocabulary (`"genuine"`, `"mixed"`, `"performed"`). The calibration script maps
current labels to legacy ones for comparison so accuracy results remain valid
against the historical artifact.

## A/B cross-check: pair provenance

### What changed

`ABTestResult` has a new `pair_provenance: str` field:

| Value | Meaning |
|-------|---------|
| `"machine-inverted"` | The B framing was generated by the model (default) |
| `"user-authored"` | Both framings were written by the analyst |

A new function `run_ab_test_with_user_task(task, user_inverted_task, ...)` lets
analysts supply both framings directly, producing a `user-authored` result.

### Why

Machine-inverted B framings share framing conventions with the model being
tested. This is useful for exploratory analysis but is not a controlled
experiment. The provenance field makes this explicit in reports.

### Migration

**CLI**: Pass `--user-inverted-task` to `robopsych crosscheck` to supply the B
framing manually.

**Python code:**

```python
from robopsych.crosscheck import run_ab_test_with_user_task

result = run_ab_test_with_user_task(
    task="Explain the architecture decision.",
    user_inverted_task="Critique the architecture decision.",
    provider=provider,
    judge_provider=judge,
    model="claude-sonnet-4-6",
    judge_model="claude-sonnet-4-6",
)
assert result.pair_provenance == "user-authored"
```
