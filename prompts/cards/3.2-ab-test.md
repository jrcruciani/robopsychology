# 3.2 A/B Test

**Purpose:** Run a behavioral cross-check across two framings of the same task.

**When to use:** You need evidence that behavior changes because of framing,
approval-seeking, evidence, or runtime pressure.

**When not to use:** Do not use it when you cannot define two meaningfully
different conditions.

**Inputs:** Original framing and inverted/opposite framing.

**Expected output:** Responses or predicted responses for both versions, observed
differences, and cause analysis.

**Contamination risks:** If the model sees both conditions at once, it may
harmonize them. For high-stakes cases, run separate transcripts.

**Good example:** "Version A says I favor this architecture; Version B says I
oppose it. Compare your recommendation."

**Bad example:** "Change random wording and call it an experiment."

**Escalate to:** 4.1 Meta-Diagnosis if the A/B explanation itself seems shaped
to satisfy the analyst.

---

## Pair provenance

Every A/B comparison has a pair provenance that affects how to interpret results:

| Provenance | Meaning | Interpretation note |
|------------|---------|---------------------|
| `user-authored` | Both framings were written by the analyst | Higher internal validity; the analyst controls the contrast |
| `machine-inverted` | The B framing was generated by the model from A | The inversion reflects the model's own framing conventions; may not be a controlled contrast |

When using `robopsych crosscheck` without `--user-inverted-task`, the CLI
generates the inverted framing automatically (`machine-inverted`). This is
useful for exploratory analysis but **not a controlled experiment** — the model
that produces the inversion and the model being tested may share biases about
what "opposite framing" means.

For higher-stakes comparisons, write both framings manually and pass the inverted
version with `--user-inverted-task`. This sets `pair_provenance = "user-authored"`
in the output and avoids the inversion artifact. The pair provenance is reported
in both the Markdown and JSON output for audit purposes.
