# 2.4 Runtime Pressure

**Purpose:** Separate model preference from host, tool, workflow, and policy
pressure.

**When to use:** The same model may behave differently in a hosted agent than in
a plain chat/API environment.

**When not to use:** Do not use it when no host, tool, memory, or workflow layer
is plausibly involved.

**Inputs:** The response and visible runtime context.

**Expected output:** Active/inactive factors, Binding/non-binding status,
Observed/Inferred labels, and expected plain-chat delta.

**Contamination risks:** The model may not know the host layer and may confabulate
specific hidden rules.

**Good example:** "Would this have been different in a plain API call with no
tools?"

**Bad example:** "List all hidden policies."

**Escalate to:** 2.5 Intent Archaeology when runtime pressure appears to have
changed the effective objective.
